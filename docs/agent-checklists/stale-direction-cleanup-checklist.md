# Stale Direction Cleanup Checklist

Before adding a new direction, ask:

* What current text does this supersede?
* What files now become misleading?
* Can the old file be deleted?
* Should it be archived?
* Should the owning currentness doc be rewritten?
* Are there generated outputs that should be ignored or removed?
* Are tests needed to prevent the stale pattern from returning?

Do not add a new current direction without resolving the old one.
