# LLM Experimentation Model

V3 exists primarily to discover LLM-centered methodologies for turning messy trader source material into useful trading intelligence.

---

## Scaffold Phase

The scaffold phase does not run live LLM calls.

This is intentional.

The scaffold phase builds the harness needed to record, validate, evaluate, compare, and synthesize future LLM experiments.

---

## Future LLM Methods

Future labs may experiment with:

- long-context LLM readers,
- chunked source-grounded LLM readers,
- multimodal/vision-language readers,
- multi-agent debates,
- example-to-principle extraction,
- contradiction mining,
- concept induction,
- formula recovery,
- source-native strategy extraction,
- model-judge evaluation,
- human-in-the-loop review,
- prompt/program optimization.

This list is illustrative, not exhaustive.

---

## What Future LLM Runs Must Record

Every future live LLM run must record:

- benchmark pack,
- source scope,
- method id,
- experiment id,
- model/provider id,
- prompt/template hash,
- model configuration,
- parameters,
- output artifact ids,
- evaluator ids,
- failures,
- negative-result value,
- research note.

---

## Proposal-Only Rule

LLM outputs are proposals until evaluated.

They do not create source truth, role support, data truth, validation, strategy evidence, financial advice, product authority, or architecture.

---

## No Hidden Learning Claims

V3 may learn through recorded artifacts, evaluation records, failure patterns, method comparisons, and research notes.

V3 may not claim that “the system learned” unless the learning is represented by explicit exported records.

---

## Negative Results

Negative LLM results are first-class outputs.

A failed method can still teach:

- what source structure it cannot handle,
- what grounding problem it creates,
- what hallucination pattern appears,
- what evaluator weakness was exposed,
- what another lab should avoid.

---

## Scaffold Fixture Disclaimer

Milestone-one fixture records are not real LLM results.

They are validation examples for the protocol.
