# V2 Benchmark Memory

V2 is used as benchmark memory and failure memory, not architecture.

---

## What V2 Contributes

V2 may contribute:

- examples of weak source grounding,
- examples of report-only confusion,
- visual/deictic blocker cases,
- formula-missing cases,
- generated artifact drift examples,
- authority drift examples,
- study card and evidence artifact examples,
- source/span/hash discipline lessons,
- negative examples of overgrown packet structure.

---

## What V2 Does Not Contribute

V2 does not contribute:

- canonical architecture,
- packet machinery,
- currentness machinery,
- product doctrine,
- shared lab runtime,
- default ingestion model,
- default extraction model,
- universal LLM-proposal-to-Python-truth law.

V2’s methods may be studied as one family of approaches, but they are not V3 authority.

---

## Benchmark Pack Rule

Every active benchmark pack must include `v2_lesson_refs`.

These references explain what V2 lesson or failure mode the benchmark preserves.

They do not import V2 architecture.
